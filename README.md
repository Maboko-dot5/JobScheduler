# JobScheduler

JobScheduler is a .NET job scheduling prototype for processing historical IoT time-series data. Jobs are submitted over HTTP, queued, processed in the background, and can run physical filtering, anomaly filtering, statistics, and PDF report generation tasks.

## Projects

- `Scheduler.Api` - ASP.NET Core API for job submission, status polling, and report download.
- `Scheduler.Client` - Console client for submitting jobs to the API.
- `Scheduler.Application` - Application orchestration, handlers, DTOs, queue, and background workers.
- `Scheduler.Domain` - Domain entities, enums, and value objects.
- `Scheduler.Infrastructure` - In-memory repositories, mock time-series data, filtering/statistics/report/email services.
- `Scheduler.Tests.Unit` - Unit tests.
- `Scheduler.Tests.Integration` - API integration tests.

## Run The API

From the repository root:

PowerShell:

```powershell
$env:ASPNETCORE_ENVIRONMENT = "Development"
dotnet run --project Scheduler.Api --urls "http://localhost:5000"
```

Git Bash:

```bash
ASPNETCORE_ENVIRONMENT=Development dotnet run --project Scheduler.Api --urls "http://localhost:5000"
```

The API will be available at:

```text
http://localhost:5000
```

Useful endpoints:

- `GET /health`
- `GET /api/jobs`
- `POST /api/jobs`
- `GET /api/jobs/{id}`
- `GET /api/jobs/{id}/report/download`

## Run The Client

Start the API first, then open a second terminal:

```bash
dotnet run --project Scheduler.Client -- "http://localhost:5000"
```

Use the project folder name as shown above. This works from Git Bash and PowerShell when run from the repository root.

The client presents an interactive menu for submitting statistics, physical filter, anomaly filter, and PDF report jobs.

### Answering The Client Menu

When the client asks for variables, do not type quote characters. Type comma-separated variable names directly:

```text
temperature,pressure
```

Do not type:

```text
'temperature,pressure'
```

The quotes would become part of the variable names and the output would show names such as `'temperature` and `pressure'`.

Example statistics job:

```text
Choose option (1-5): 1
Enter job name (default: 'Test Job'): TJ1
Enter plant ID (default: 'PlantA'): Plant-001
Enter variables (comma-separated, default: 'temperature,pressure'): temperature,pressure
Enter days back (default: 1): 2
Enter statistics window (0=Shift, 1=Daily, 2=Weekly, default: 1): 2
```

Example PDF report job:

```text
Choose option (1-5): 4
Enter job name (default: 'Test Job'): TJ1
Enter plant ID (default: 'PlantA'): Plant-001
Enter variables (comma-separated, default: 'temperature,pressure'): temperature,pressure
Enter days back (default: 1): 3
```

For PDF report jobs, the task result includes a full download URL such as `http://localhost:5000/api/jobs/{jobId}/report/download`, which can be Ctrl+clicked from most terminals.

## Submit A Job Manually

Example PDF report job:

```powershell
$jobId = [guid]::NewGuid()
$body = @{
  jobId = $jobId
  name = "Daily report"
  primaryTaskType = 3
  plantId = "PlantA"
  variables = @("temperature", "pressure")
  requestedTimeRange = @{
    startUtc = (Get-Date).ToUniversalTime().AddDays(-1)
    endUtc = (Get-Date).ToUniversalTime()
  }
  requestedBy = "manual-test"
  tasks = @(
    @{
      taskType = 3
    }
  )
} | ConvertTo-Json -Depth 5

Invoke-RestMethod -Method Post -Uri "http://localhost:5000/api/jobs" -ContentType "application/json" -Body $body
```

Poll the job:

```powershell
Invoke-RestMethod "http://localhost:5000/api/jobs/$jobId"
```

Download the generated report after the job completes:

```powershell
Invoke-WebRequest "http://localhost:5000/api/jobs/$jobId/report/download" -OutFile ".\report.pdf"
```

## Runtime Notes

- Persistence is in-memory. Jobs, statuses, queued emails, and generated reports are lost when the API process stops.
- The time-series database is represented by `MockTimeSeriesRepository`, which generates deterministic sample data.
- Physical filtering uses static per-variable rules.
- Anomaly filtering uses a z-score based implementation.
- Statistics support shift, daily, and weekly windows.
- PDF reports are generated by the infrastructure report generator and can be downloaded from the report endpoint.
- Email delivery uses the current console/logging implementation. No SMTP credentials or external mail server are required to build, run, or test the application.
